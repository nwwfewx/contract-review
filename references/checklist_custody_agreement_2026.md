# Custody Agreement Supplement Checklist (2026) 托管协议条款增补清单（2026 新规）

> 适用触发：合同文本出现 `托管协议` / `托管合同` / `托管银行` / `托管人职责` / `投资监督` / `估值复核`。
>
> 叠加方式：本清单为 overlay，不改变 `canonical_mode`；在 `general` 或 `asset_mgmt:*` 基础清单上叠加。
>
> 法规基准：`商业银行托管业务监督管理办法（试行）`（2025-12-12 发布，2026-02-01 起施行）。
>
> 管理人视角：同时实现两件事
> 1. 锁定托管人“可履职且必须履职”的职责边界；
> 2. 防止免责条款扩大到托管人可控制事项。

## 五大制度革新核对（先做）

- [ ] 法律效力层级已更新：托管协议依据由“行业自律文件”升级为“部门规章 + 上位法体系”。
- [ ] 职责边界已按“正面清单 + 负面清单”重述，不再使用笼统表述。
- [ ] 履职标准已细化到可执行动作（账户命名、估值数据来源、投资监督流程）。
- [ ] 涉及公募/私募基金产品的，已补充“双重监管框架”适配条款（金融监管总局 + 证监体系）。
- [ ] 权责平衡已验证：托管人援引免责的同时，未削弱其法定履职义务。

## 使用前提（门禁）

- [ ] 确认协议对象属于商业银行托管业务，不与一般“资金监管协议”混用口径。
- [ ] 确认合同版本优先级（签署版 > 最新补充协议 > 草稿）后再进入条款比对。
- [ ] 对法规依据仅使用可核验表述；条号不确定时用“关键词锚点 + `【需人工复核】`”。

## 十大条款增补方向（管理人视角）

### 1) 依据条款与效力层级
- [ ] 明确协议依据包含《商业银行托管业务监督管理办法（试行）》及适用监管规则。
- [ ] 约定“法律法规/监管规定更新时，协议自动从其高位规则并启动修订”。
- [ ] 存量业务补充整改安排与过渡期限（新规施行后整改要求）明确。
- [ ] 对公募/私募基金等双重监管产品，补充“同时遵守业务主管部门规则”的衔接表述。

### 2) 托管职责正面清单（服务范围）
- [ ] 服务范围至少覆盖：账户开立、财产保管、清算交割、会计核算、资产估值、信息披露、投资监督。
- [ ] 托管职责不使用“尽力而为但不承担任何审查义务”等空泛表述替代实质义务。
- [ ] 涉及非标资产（非标债权/未上市股权等）时，约定托管人评估后承接及风险管控义务。

### 3) 负面清单与免责边界
- [ ] 明确托管人不承担信用风险、市场风险，不参与投资决策。
- [ ] 明确不得约定托管人提供保本保收益、垫资、流动性支持、融资承诺。
- [ ] 免责条款不得覆盖托管人自身履职错误、过失或违反明确指令导致的损失。
- [ ] 对“资金来源合法性、投资项目真实性”等非托管职责表述不得反向转嫁为管理人绝对责任。
- [ ] 托管人新增免责不得突破《办法》列举范围；出现扩张性免责时标记 `P0`。

### 4) 账户开立、命名与财产独立
- [ ] 明确一产品一账户（资金账户、证券账户等）和专户专用要求。
- [ ] 账户命名规则体现“开户主体 + 产品名称”，保持与产品文件一致。
- [ ] 约定托管财产与托管人自有财产、其他产品财产相互独立及破产隔离原则。

### 5) 对账、结算与资金划拨指令机制
- [ ] 对账机制明确，频率不低于半年度，并约定差异处理时限与升级路径。
- [ ] 结算条款明确双方在清算交收和风控中的职责分工。
- [ ] 资金划拨以明确指令与授权为前提；无授权不得擅自划拨或资产处置。
- [ ] 异常交易/无法拒绝执行时，托管人提示义务、留痕义务和通知链条明确。
- [ ] 清算交割责任岗位、授权链条、替岗机制和审计留痕要求明确。

### 6) 会计核算与估值条款
- [ ] 每只产品单独建账、单独核算的责任分工和方法写清。
- [ ] 可从第三方获取可靠数据时，优先使用第三方数据。
- [ ] 仅能依赖管理人数据时，数据来源、校验边界和免责条件必须写明。
- [ ] 估值对象、估值原则、方法、时点、差错处理、估值分歧后的披露动作完整。

### 7) 信息披露职责与复核边界
- [ ] 信披职责、内容、方式、时点与触发条件明确。
- [ ] 托管人发现重大事项时，提示管理人披露的时效要求明确。
- [ ] 需托管人复核的披露信息，约定“复核范围仅限托管人可获取数据”，并在披露中标注复核边界。

### 8) 投资监督条款（可履职前提）
- [ ] 监督事项、监督内容、监督方式可执行，避免笼统“全面监督”。
- [ ] 监督前提写明“托管人已充分获取必要数据”。
- [ ] 不得约定或承诺托管人提供无法有效履职的监督服务。
- [ ] 发现违规投资时，拒绝执行、无法拒绝时的提示和报告路径明确。
- [ ] 条款明确托管人在能力范围内采取必要合理措施，不得以“数据不足”笼统豁免全部监督义务。

### 9) 重大风险事件、终止与移交
- [ ] 触发报告事件（违规投资拒不纠正、估值披露异常、侵占挪用、失联等）在合同中可操作化。
- [ ] 管理人拒不履约、资质取消、解散撤销等情形下，托管人保护财产和通知相关方的义务明确。
- [ ] 托管终止、托管机构变更时，资料和资产移交流程、时限、责任清晰。
- [ ] 托管业务资料保存期限不少于 10 年并写入合同。
- [ ] 托管人单方终止权（含不当营销情形）应配套整改通知、合理补救期和连续性过渡安排。

### 10) 违约责任、不当营销与争议解决
- [ ] 违约责任按控制能力分配，不出现“无责但享有广泛免责”的失衡结构。
- [ ] 赔偿范围、损失计算口径、举证责任与证据保全义务明确。
- [ ] 对不属于托管职责范围的追偿/诉讼/破产程序义务仅在法律明确要求时承担。
- [ ] 争议解决条款覆盖送达、保全配合、持续履行与应急机制。
- [ ] “不当营销”定义边界清楚，不将依法合规披露或事实性展示误认定为违约。
- [ ] 不当营销责任应与过错、影响范围和纠正机制挂钩，避免无限责任或自动解约。

## 监管锚点关键词（便于证据定位）

| 审核主题 | 建议锚点关键词 |
|---|---|
| 正面职责 | “账户开立、财产保管、清算交割、会计核算、资产估值、信息披露、投资监督” |
| 负面职责 | “不得承担以下职责” |
| 禁止行为 | “不得有以下行为” |
| 账户管理 | “分别开立资金账户、证券账户” |
| 对账频率 | “对账频率应至少每半年一次” |
| 划拨授权 | “未获得明确指令和授权，不得擅自进行资金划拨” |
| 第三方数据 | “应当主要使用第三方数据进行会计核算” |
| 投资监督边界 | “不得提供或承诺提供无法有效履职的投资监督服务” |
| 违规投资处置 | “违反法律法规或合同约定进行投资的，应当拒绝执行” |
| 不当营销 | “借助托管银行品牌、声誉开展不当营销宣传” |
| 保存期限 | “保存10年以上” |

## P0 红线（触发即升级）

- [ ] 托管协议将托管人法定职责整体弱化为“协助/配合”，未设明确履职义务。
- [ ] 托管协议出现保本保收益、垫资、流动性支持、融资承诺安排。
- [ ] 账户不独立或账户命名无法对应具体产品，存在资金混同风险。
- [ ] 对账机制低于半年度或无差异处理时限。
- [ ] 投资监督条款超出可履职边界但又约定托管人免责。
- [ ] 估值/核算仅依赖管理人数据，且未约定数据来源和免责边界。
- [ ] 违约责任极度失衡，管理人承担绝对责任而托管人无实质责任。
