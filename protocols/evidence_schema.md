# Evidence Schema 证据链字段规范

> 每条审查意见必须包含完整证据链

## Required Fields 必填字段

| 字段 | 字段名 | 类型 | 说明 | 示例 |
|------|--------|------|------|------|
| 条款位置 | `clause_location` | string | 章节号 + 条款号 | "第三章 第12条 第2款" |
| 原文摘录 | `original_text` | string | 原始条款文本（限200字） | "甲方应配合省一级分支机构..." |
| 风险描述 | `risk_description` | string | 风险问题的简明描述 | "配合义务范围过窄" |
| 触发后果 | `risk_consequence` | string | 风险触发后的具体后果 | "可能导致监管检查时无法满足要求" |
| 审查依据 | `legal_basis` | string | 法律/监管/制度依据 | "《反洗钱法》第43条" |
| 依据等级 | `basis_level` | enum | 依据效力层级 | `L1_法律` |
| 修改建议 | `suggested_revision` | string | 可直接替换的修改文本 | "配合设区的市级以上分支机构..." |
| 优先级 | `priority` | enum | 风险优先级 | `P0_阻断` |
| 对象范围 | `entity_scope` | string | 结论适用主体与层级 | "甲方分支机构/总公司" |
| 比较口径 | `comparison_basis` | string | 指标语义+时间窗+场景+可比性说明 | "同类合同, 2025Q4, 投资端" |
| 因果链 | `causality_chain` | string | 事实->机制->中介->边界 | "条款缺口->执行延迟->违约触发->仅在..." |
| 适用边界 | `scope_boundary` | string | 不成立条件与降级说明 | "若补充协议明确，则降级P1->P2" |

## Priority Levels 优先级定义

| 等级 | 标识 | 含义 | 处理要求 |
|------|------|------|----------|
| P0 | `P0_阻断` | 一票否决项 | 必须解决，否则终止审查 |
| P1 | `P1_重要` | 重大风险 | 强烈建议修改 |
| P2 | `P2_建议` | 可优化项 | 建议修改但不阻断 |
| P3 | `P3_备注` | 提示信息 | 仅供参考 |

## Basis Levels 依据等级

| 等级 | 标识 | 来源 | 强制性 |
|------|------|------|--------|
| L1 | `L1_法律` | 全国人大及其常委会 | 强制 |
| L2 | `L2_行政法规` | 国务院 | 强制 |
| L3 | `L3_规章` | 部委/地方政府 | 管理性 |
| L4 | `L4_规范性文件` | 监管机构 | 指导性 |
| L5 | `L5_司法解释` | 最高法/最高检 | 裁判依据 |
| L6 | `L6_行业惯例` | 行业协会/通行做法 | 参考性 |
| L7 | `L7_内部制度` | 公司内部规定 | 内部约束 |
| L8 | `L8_合同逻辑` | 上下文一致性 | 合理性 |

## Output Example 输出示例

```yaml
finding:
  id: 1
  clause_location: "第四章 第15条"
  original_text: "乙方应在收到通知后5个工作日内完成整改"
  risk_description: "整改期限过短，可能无法完成复杂整改"
  risk_consequence: "超期未整改可能触发违约条款，导致解约"
  legal_basis: "合同法诚实信用原则"
  basis_level: L8_合同逻辑
  suggested_revision: "乙方应在收到通知后15个工作日内完成整改，复杂事项可协商延期"
  priority: P2_建议
  entity_scope: "乙方运营团队"
  comparison_basis: "同类型合同整改周期, 2025, 运营场景"
  causality_chain: "期限过短->排期失败->逾期违约->仅在并行整改任务高峰成立"
  scope_boundary: "若新增延期机制, 风险降级"
```
