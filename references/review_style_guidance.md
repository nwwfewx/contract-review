# Review Style Guidance

Use this guidance only when the user explicitly signals a preference.
Default is `平衡`.

## Styles

### 1) 效率优先（业务友好/领导口径）
Focus on execution viability and time-sensitivity.
- Prefer **clarifying terms** over adding new obligations.
- Avoid adding pre-approval steps unless strictly necessary.
- If approvals are needed, add **time limits** or **post-reporting** options.
- Avoid new “提示/提醒义务” unless required by regulation.
- For time-sensitive assets, prioritize **window-friendly processes**.
 - Use the “four-piece balance”: **time limits + default rule + post-reporting + passive-event carveout**.
 - Do NOT omit compliance bottom lines (资质/偿付能力/监管红线/关联交易时限/资产隔离).

### 2) 合规闭环（风险最小化）
Focus on auditability and full process closure.
- Add explicit reporting/approval/record-keeping steps.
- Require clear thresholds, timelines, and escalation triggers.
- Prefer explicit written confirmations.

### 3) 平衡（默认）
Moderate between compliance and execution:
- Add only high-impact controls.
- Keep approval steps minimal but time-bounded.
- Use “先执行后补报” for window-sensitive trades where reasonable.
 - Keep compliance bottom lines intact.
