# Veto Gate

用途：一票否决与阻断风险交叉检查。  
执行方式：对照 `protocols/veto_list.md` 与当前项目结论。

## Gate Items
- [ ] V1 是否存在触发否决项且被误降级处理
- [ ] V2 资金用途是否可监测、可穿透（不可穿透默认高风险）
- [ ] V3 授权链是否完整（董事会/股东会/授权文件）
- [ ] V4 法人独立性是否存在重大瑕疵（SPV 资产混同/空壳）
- [ ] V5 增信与担保是否具备可执行路径
- [ ] V6 上市公司/金融机构约束是否完成必要披露与审批判断
- [ ] V7 若业务坚持推进，是否已给“留痕要求 + 决策签收责任”

## PASS Criteria
1. 触发 V1-V6 任一重大问题时，结论至少为 P0/P1，不得默认为 P2/P3。  
2. 所有“坚持推进”场景必须写明前置条件与留痕动作。  
3. 未通过项必须显式列入交付风险摘要。
