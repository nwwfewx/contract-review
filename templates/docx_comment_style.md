# DOCX 批注措辞模板

## 目标
- 只写交付版措辞，不复述条款位置/原文摘录
- 可执行、短句直达，不堆叠解释
- 一条批注对应一个动作（多动作拆分多条）
- 不写“前置条件/推进前置条件”

## 动作词
- 删除 / 新增 / 合并 / 移动 / 改为 / 调整 / 补充 / 明确 / 限定 / 统一口径 / 增加时限 / 增加例外 / 前置条件

## 基本结构
- `动作 + 目标 + 改法/范围`
- 优先使用“改为/删除/新增/合并/移动”这种明确可执行的指令
- 需要替换文字时，用引号明确新旧文本

## 示例
- `将“……(旧)”改为“……(新)”。`
- `删除“……(旧)”。`
- `新增“……(新)”。`
- `合并本条与“……”，改为“……(新)”。`
- `移动“……(旧)”至“……”之后。`

## 提示型批注（非修改）
- 以“提示：/关注：”开头，简短点出风险或执行注意事项
- 不与修改型内容混写

## 禁止
- 不写条款位置、不写原文摘录
- 不照搬表格字段
- 不把“风险描述/触发后果”原样塞进批注
- 不写“前置条件/推进前置条件”
