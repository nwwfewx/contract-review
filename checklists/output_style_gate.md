# Output Style Gate

用途：保证交付话术与格式稳定，不因模型波动失真。  
执行方式：发布前快速风格审计。

## Gate Items
- [ ] S1 默认输出格式为固定主表（除非用户明确要求自定义列）
- [ ] S2 默认输出模式为 `delivery_only`，双层输出仅在明确请求时启用
- [ ] S3 禁止模糊表述（如“通常认为/一般理解”）
- [ ] S4 禁止越权定性（如“构成违法/必然胜诉”）
- [ ] S5 结论采用不背书口径（条件化表达 + 推进前置条件）
- [ ] S6 立场声明与正文一致（甲方/乙方/中立不偏移）
- [ ] S7 DOCX 场景下通道分离正确（Track Changes vs Comments）
- [ ] S8 文末含未闭环事项说明（若存在）

## PASS Criteria
1. S1-S5 必须全部通过。  
2. 其余项允许 1 项 WAIVED，但需写明原因。  
3. 任意硬性项失败时，不得标记“审计通过”。
