# DOCX 修订 & 批注措辞规范

## 交付模式判定

- 命中“分毫不差/沿用最新版/交付件/版式锁定”时，切到 `layout-lock`。
- 其他场景默认 `revision-docx`。

## `layout-lock` 版式锁定规则

- 以同类型项目最新定稿 `.doc/.docx` 为母版，先另存为，再在母版内原位改文。
- 原位改文优先使用 LibreOffice/Word/WPS 原生文档引擎，禁止 markdown/docx 重排链路。
- 保留母版现有页眉页脚、分页、目录与表格样式骨架，不做整稿重建。
- 语义规则不变：修改型动作走 Track Changes，提示型动作走 Comments。
- 交付前必须完成旧项目残留词反查，并抽检封面、目录、核心表格。

## 双通道规则

| 操作类型 | 通道 | Word 显示效果 |
|----------|------|---------------|
| 改为/调整/修订 | **Track Changes** | 删除线（旧）+ 下划线（新） |
| 删除 | **Track Changes** | 红色删除线 |
| 新增/补充/明确/限定/统一/约定 | **Track Changes** | 彩色下划线 |
| 合并/移动 | **Comments（批注）** | 批注栏文字 |
| 提示/关注/风险说明 | **Comments（批注）** | 批注栏文字 |

## Track Changes 规范
- 修订作者：`WynnSing`
- 修订精度：定位到 run 级别（段落内的文本片段），非段落级
- 替换操作：产生 `<w:del>` + `<w:ins>` 配对
- 删除操作：产生 `<w:del>` 包裹
- 插入操作：产生 `<w:ins>` 包裹
- 格式保持：新插入内容继承原 run 的格式属性（`<w:rPr>`）

## 批注规范（仅提示型）
- 以 `提示：` 或 `关注：` 开头
- 简短点出风险或执行注意事项
- 不与修改操作混写
- 不写条款位置、不写原文摘录
- 不照搬表格字段

## 定位优先级
1. "原文摘录"片段 → 段落 → run
2. 条款号仅匹配段首（如"第9条/9.1 …"）
3. 附件范围内定位
4. 无法匹配 → 跳过 + 输出未命中清单

## 禁止
- 不写"前置条件/推进前置条件"
- 不把"风险描述/触发后果"原样塞进修订或批注
- 不用裸数字匹配条款号
